# BRS-01: Screen 1 - Landing Dashboard & Project Initialization

## 1. Introduction (บทนำ)
ความต้องการในเอกสารฉบับนี้กำหนดเกณฑ์และข้อกำหนดเชิงธุรกิจสำหรับ **หน้าจอที่ 1 (Screen 1): Landing Dashboard & Project Initialization** ของระบบ **IREBA** เพื่อเป็นจุดต้อนรับผู้ใช้งาน แสดงประวัติการดำเนินการสกัดความต้องการในลักษณะของโครงการ (Dashboard) และฟอร์มสร้างโครงการใหม่ โดยอ้างอิงการจัดสไตล์ตามชุดตัวแปรของ [BRS-UX-CSS-Design-System.md](file:///d:/development/IRIS-Training/IRIS-Prototype/Requirements/business-requirements/BRS-UX-CSS-Design-System.md)

## 2. Business Requirements & UI Specs (ความต้องการทางธุรกิจ)

### 2.1 UI Component & Layout Specifications (การจัดวางและส่วนประกอบหน้าจอ)
การออกแบบหน้าจอต้องมีลักษณะสะอาดตา (Clean UI) ใช้พื้นที่ว่างอย่างเหมาะสม และแบ่งการนำเสนอตามสถานะข้อมูลออกเป็น 2 มุมมองหลัก ดังนี้:

#### 2.1.1 มุมมองแดชบอร์ดโครงการ (Dashboard View) - มุมมองเริ่มต้น
ในกรณีที่มีประวัติโครงการบันทึกอยู่ในฐานข้อมูลจำลอง หน้าจอแรกจะต้องเรนเดอร์ในมุมมองนี้โดยประกอบไปด้วย:
* **Global Header:** แสดงชื่อระบบ **"IREBA"** ขนาดเด่นชัดสีน้ำเงินหลัก `--color-primary-navy`
* **Dashboard Container:** การ์ดเนื้อหาขนาดใหญ่ขอบโค้งมน `--radius-lg` และเงาฟุ้งเบาบาง `--shadow-soft`
  * **Header Area:** แสดงจำนวนโครงการทั้งหมดในระบบ และมีปุ่ม **"+ New Project"** อยู่มุมบนขวา (สีน้ำเงินพื้นผิวทึบ สลับเป็นครามเมื่อโฮเวอร์) เพื่อใช้เรียกเปิดฟอร์มสร้างโครงการใหม่ในแบบ Modal
  * **Project List Table/Grid:** ตารางหรือกริดรายการโครงการเก่า เรียงลำดับตามเวลาสร้างล่าสุด ประกอบด้วยคอลัมน์:
    * **Project ID:** รหัสโครงการ (เช่น `PRJ-2026-001`) แสดงด้วยตัวอักษรหนา
    * **Project Name:** ชื่อโครงการประกันภัย พร้อมคำอธิบายย่อ
    * **Created Date:** วันเวลาที่สร้างโครงการ
    * **Status Pill Badges:** ป้ายวงกลมขอบมนแสดงสถานะของแต่ละโครงการ โดยแยกสีตาม Design System ดังนี้:
      * `initialized` : ป้ายสีน้ำเงิน/ครามอ่อน (`--color-primary-navy` หรือครามจาง)
      * `documents_uploaded` : ป้ายสีส้มหรือสีเหลืองทองจาง
      * `analyzed` : ป้ายสีม่วงจาง
      * `approved` : ป้ายสีเขียวอ่อนจาง (`--color-success-green` จาง)
    * **Actions:** ปุ่มหรือลิงก์คำสั่ง `"เปิดดำเนินการ ➜"` เพื่อกดเข้าสู่ขั้นตอนการทำงานล่าสุดของโครงการนั้นๆ
* **Footer (เลขรุ่นและประวัติระบบ):**
  * บริเวณขวาล่างแสดงป้ายรุ่นแอปพลิเคชันระบุว่า `"IREBA App Version: v3.0.0"` รองรับการกดเพื่อดึงประวัติ Changelog Popover

#### 2.1.2 หน้าจอว่างเปล่า (Empty State View)
ในกรณีที่เปิดระบบขึ้นมาครั้งแรกหรือไม่มีโครงการใดๆ บันทึกอยู่ในไฟล์ `projects.json` ระบบจะต้องเรนเดอร์เป็นหน้าจอว่างเปล่าสวยงาม:
* แสดงข้อความแนะนำขนาดใหญ่กึ่งกลางหน้าจอ: *"ยังไม่มีโครงการจัดตั้งในระบบ"*
* มีรายละเอียดแนะนำการใช้งานและปุ่มกดเด่นชัดกึ่งกลางหน้าจอ: **"เริ่มสร้างโครงการแรกของคุณ ➜"** (ปุ่มสีน้ำเงินหลัก) เมื่อกดแล้วจะทำการเปิดฟอร์มสร้างโครงการใหม่ขึ้นมา

#### 2.1.3 ฟอร์มสร้างโครงการใหม่ (Create Project Modal)
แสดงเมื่อผู้ใช้งานคลิกปุ่ม "+ New Project" หรือปุ่ม "เริ่มสร้างโครงการแรกของคุณ ➜" โดยเปิดเป็นหน้าต่างลอยลื่นไหลกึ่งกลางหน้าจอ:
* **Project Name Input:** ช่องรับข้อมูลตัวหนังสือจำเป็นต้องกรอก (3-100 ตัวอักษร)
* **Project Description Input:** TextArea รายละเอียดโครงการ
* **Action Buttons:** ปุ่ม `"Create Project & Continue"` และปุ่ม `"ยกเลิก"`

### 2.2 System Processing & Navigation Rules (กฎการประมวลผลระบบและการนำทาง)
* **การบันทึกโครงการใหม่:** เมื่อกรอกข้อมูลโครงการสำเร็จ ระบบบันทึกประวัติลงไฟล์ `projects.json` โดยมีสถานะตั้งต้นเป็น `"initialized"` แล้วนำทางเปลี่ยนหน้าจอไปยัง **Screen 2: Document Ingestion** โดยอัตโนมัติ
* **ตรรกะการคลิกกลับเข้าไปโครงการเก่า (Resume Extract History):** เมื่อผู้ใช้คลิกปุ่มนำทางกลับเข้าโครงการเก่าที่หน้า Dashboard ระบบจะส่งผู้ใช้งานไปยังขั้นตอนการสกัดงานล่าสุด โดยอ้างอิงตามฟิลด์ `status` ของโครงการนั้นๆ:
  1. หากสถานะคือ **`initialized`** -> นำทางไปยัง **Screen 2 (Document Ingestion)**
  2. หากสถานะคือ **`documents_uploaded`** -> นำทางไปยัง **Screen 3 (AI Output Config)**
  3. หากสถานะคือ **`analyzed`** หรือ **`approved`** -> นำทางไปยัง **Screen 4 (Result Display & Export)**

## 3. Data Schema Specifications (โครงสร้างข้อกำหนดข้อมูล)

### 3.1 Flat-File Database Schema (`projects.json` structure)
สถานะของโปรเจกต์จะถูกเปลี่ยนผ่านตามรูปด้านล่าง:
```json
[
  {
    "projectId": "PRJ-2026-001",
    "projectName": "FutureShield 99 Implementation",
    "description": "โครงการวิเคราะห์และสกัดข้อมูลแบบประกันภัยเพื่อการพัฒนาและทดสอบระบบ",
    "createdAt": "2026-06-12T04:55:07Z",
    "status": "initialized",
    "documents": {
      "productSpec": null,
      "pricingValuation": null,
      "compliance": null
    },
    "extractedRequirements": null
  }
]
```

## 4. Test-Driven Development (TDD) & UAT Scenarios

### 4.1 Gherkin Test Scenarios

#### Scenario 1: การลงทะเบียนสร้างโครงการใหม่สำเร็จและเข้าสู่หน้าถัดไป (UAT: Initial Project Creation)
* **Given** ผู้ใช้งานอยู่ในหน้าแรกและเปิดกล่อง Modal สร้างโครงการขึ้นมา
* **When** ผู้ใช้งานกรอกชื่อโครงการเป็น `"FutureShield 99"` และคลิกปุ่ม `"Create Project & Continue"`
* **Then** ระบบต้องทำการสร้างรหัสโครงการรูปแบบตัวอักษร `"PRJ-2026-001"` และบันทึกลง `projects.json`
* **And** ระบบต้องนำทางผู้ใช้งานเปลี่ยนสเตทไปยัง Screen 2 (นำเข้าเอกสาร) เพื่อดำเนินงานต่อ

#### Scenario 2: การเปิดรายละเอียดประวัติการอัปเดตแอปพลิเคชันผ่าน Footer (UAT: Open Changelog Popup)
* **Given** ผู้ใช้งานเปิดหน้าจอใดๆ ของระบบ IREBA
* **When** ผู้ใช้งานคลิกข้อความเลขรุ่น `"IREBA App Version: v3.0.0"` ที่บริเวณ Footer
* **Then** หน้าจอต้องแสดงกรอบข้อความ Changelog Popover แสดงรายละเอียดฟีเจอร์ทั้ง 5 ข้อของรุ่น
* **And** เมื่อคลิกพื้นที่อื่นภายนอกกล่อง หรือกดปุ่มปิด กล่องรายละเอียดต้องหดหายไป

#### Scenario 3: การเรนเดอร์แดชบอร์ดโครงการและการเข้าทำงานต่อ (UAT: Dashboard View & Resume Flow)
* **Given** ระบบทำการจัดเก็บโครงการที่มีสถานะ `"documents_uploaded"` ไว้ใน `projects.json`
* **When** ผู้ใช้งานเปิดหน้าแรกของระบบ IREBA
* **Then** ระบบต้องเรนเดอร์แสดงตารางรายการโครงการที่พบ และไม่แสดงหน้า Empty State
* **And** เมื่อผู้ใช้งานคลิกปุ่ม `"เปิดดำเนินการ ➜"` บนแถวของโครงการนั้น ระบบต้องนำทางผู้ใช้ไปสู่ Screen 3 (สกัดวิเคราะห์ AI) โดยทันทีตามสถานะ

#### Scenario 4: การเรนเดอร์หน้าจอ Empty State เมื่อเปิดแอปพลิเคชันครั้งแรก (UAT: Empty State Rendering)
* **Given** ฐานข้อมูลโครงการ `projects.json` ไม่มีรายการข้อมูลโครงการใดๆ อยู่เลย
* **When** ผู้ใช้งานเปิดหน้าแรกของระบบ IREBA
* **Then** หน้าจอหลักต้องแสดงข้อความแบนเนอร์เด่นชัดระบุว่า `"ยังไม่มีโครงการจัดตั้งในระบบ"`
* **And** ต้องแสดงปุ่มคำสั่งสีน้ำเงินระบุคำว่า `"เริ่มสร้างโครงการแรกของคุณ ➜"`

## 5. Expected Output Summary (สรุปผลลัพธ์ที่ต้องการ)
1. หน้าแรกของเว็บเป็นระบบแดชบอร์ดโครงการ แสดงตารางประวัติ และป้ายสถานะแยกสีชัดเจน
2. กล่องควบคุมสร้างโครงการใหม่ปรับเปลี่ยนรูปแบบการแสดงผลเป็น Modal ลอยลื่นไหล
3. ระบบตรรกะนำทางย้อนหลังกลับเข้าสู่ขั้นตอนล่าสุดของโครงการเก่าสำเร็จอย่างถูกต้อง
