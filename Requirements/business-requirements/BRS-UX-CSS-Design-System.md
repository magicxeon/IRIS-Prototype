# BRS: UX & CSS Design System (IREBA)

## 1. Global System Identity (อัตลักษณ์ของระบบ)
* **ชื่อระบบอย่างเป็นทางการ:** Insurance Requirement Extraction for Business Analyze
* **ตัวย่อชื่อระบบ:** **IREBA**
* **แนวคิดการออกแบบหลัก (UX Core Principles):**
  * **Clean & Spacious Layout:** เน้นการใช้พื้นที่ว่าง (Whitespace) เพื่อความสบายตา ข้อมูลไม่หนาแน่นจนเกินไป จัดแนวขอบเรียงอย่างสวยงามเป็นระเบียบ
  * **Professional & Corporate Style:** ออกแบบสไตล์น่าเชื่อถือแบบมืออาชีพโดยใช้คู่สีสุภาพและเงาสะท้อนกระจกฝ้า (Glassmorphism) สำหรับสถานะการรอคอยหรือการแสดงสถานะพิเศษ
  * **User-Friendly Navigation:** มีแถบนำทางบอกขั้นตอนการทำงานแบบลำดับขั้นตอน (Step Progress Indicator) ที่เข้าใจง่าย (Screen 1 -> Screen 2 -> Screen 3 -> Screen 4)

---

## 2. Typography Specification (ข้อกำหนดด้านตัวอักษร)
เพื่อความชัดเจน สวยงาม และอ่านภาษาไทยได้อย่างถูกต้องเป็นสากล ระบบจะบังคับใช้ฟอนต์เดี่ยวกันทั้งระบบดังนี้:
* **Google Font:** **Kanit** (ฟอนต์ประเภท Sans-serif ไม่มีหัว มีความร่วมสมัยและเป็นระเบียบ)
* **การนำเข้า (Font Loading Requirement):**
  * ระบบจะต้องนำเข้าฟอนต์ผ่าน Google Fonts Web API ด้วยลิงก์:
    `https://fonts.googleapis.com/css2?family=Kanit:wght@300;400;500;700&display=swap`
  * กำหนดสไตล์ใน CSS หลักให้ประยุกต์ใช้ฟอนต์ `Kanit` และรองรับอักษรสำรอง `sans-serif` ทั่วทั้งหน้าจอ

## 3. CSS Design Tokens (โทนสีและสไตล์อ้างอิงเพื่อการแก้ไขที่ง่าย)
เพื่อประโยชน์ในการแก้ไขหน้าตาโทนสีและการจัดวางภายหลัง วิศวกรซอฟต์แวร์ (SE) จะต้องเขียนจัดระเบียบสไตล์ชีท (Main CSS File) โดยแยกตัวแปรโทเค็น (CSS Custom Properties) ออกมาไว้ที่หัวไฟล์ ดังนี้:

### 3.1 CSS Custom Variables (ชุดตัวแปรสีและขอบเงา)
* `--font-family-sans`: `'Kanit', sans-serif;` (ฟอนต์หลักระบบ)
* `--color-primary-navy`: `#1e3a8a;` (สีน้ำเงินหลักของแบรนด์ ใช้กับ Header และปุ่มยืนยันหลัก)
* `--color-secondary-indigo`: `#4f46e5;` (สีครามเข้ม ใช้สลับเฉดและสถานะการเลือก)
* `--color-background-slate`: `#f8fafc;` (สีพื้นหลังหลักของเว็บไซต์ สะอาดและสว่าง)
* `--color-surface-white`: `#ffffff;` (สีพื้นผิวการ์ดและกล่องข้อความเด่น)
* `--color-text-dark`: `#0f172a;` (สีตัวอักษรหลัก สี Slate เกือบดำทำให้อ่านง่ายขึ้น)
* `--color-border-light`: `#e2e8f0;` (สีเส้นขอบเบาบาง สะอาดตา)
* `--color-success-green`: `#10b981;` (สีเขียวสำหรับไฟล์อัปโหลดผ่าน หรือความคุ้มครองผ่าน)
* `--color-error-red`: `#ef4444;` (สีแดงเตือน Validation Error)
* `--radius-lg`: `1rem;` (ขอบการ์ดโค้งมนมนุ่มนวล 16px)
* `--radius-md`: `0.5rem;` (ขอบช่องอินพุตและปุ่มโค้งมน 8px)
* `--shadow-soft`: `0 4px 6px -1px rgb(0 0 0 / 0.05), 0 2px 4px -2px rgb(0 0 0 / 0.05);` (เงาฟุ้งเบาบางพรีเมียม)

## 4. App Version & Feature List Dropdown (เลขรุ่นแอปพลิเคชันและรายการระบบ)

ระบบจะมีการแสดงป้ายเวอร์ชันแอปพลิเคชันไว้บริเวณส่วนล่าง (Footer) เพื่อให้ผู้ใช้ตรวจทาน และสามารถกดเปิดหน้าต่างขยายดูความสามารถของแอปพลิเคชันได้โดยตรง:

### 4.1 UI Component & Behavioral Specification
* **ป้ายแสดงเลขเวอร์ชัน (Version Badge):**
  * แสดงข้อความตัวหนังสือขนาดเล็กสีเทาหม่นระบุว่า `"IREBA App Version: v3.0.0"` อยู่กึ่งกลางหรือขวาล่างของหน้าจอ Footer ทุกหน้าจอ
  * ป้ายเลขเวอร์ชันนี้จะต้องสามารถตอบสนองต่อการชี้เมาส์ (Hover Effect - แสดงสัญลักษณ์ Cursor Pointer) และตอบสนองต่อการคลิก
* **หน้าต่างฟีเจอร์เวอร์ชัน (Changelog Popover/Modal):**
  * เมื่อผู้ใช้งานคลิกที่เลขเวอร์ชัน ระบบต้องเปิดหน้าต่าง Popover ลอยขึ้นมา หรือแสดงกล่องข้อความ Modal แบบสะอาดตา
  * ภายในแสดงรายการฟีเจอร์ปัจจุบันของเวอร์ชัน `v3.0.0` ดังนี้:
    * **[Feature 1] Screen-oriented Navigation Flow:** ลำดับการกรอกและแสดงผลตามหน้าจอ 1 ถึง 4
    * **[Feature 2] Flat-file Project Storage:** บันทึกข้อมูลและสถานะการอัปโหลดอัตโนมัติลงดิสก์ `projects.json`
    * **[Feature 3] Gemini API Context Ingestion:** รองรับการส่งไฟล์ข้อกำหนดและโครงสร้างคำสั่งวิเคราะห์ผ่าน API Context Window
    * **[Feature 4] Actuarial Math Validation Engine:** สกัดสูตรคำนวณเบี้ยประกัน ตารางอัตราเบี้ยรายปี และตารางมูลค่าเวนคืนกรมธรรม์ออกมาเป็นรูปธรรม
    * **[Feature 5] Markdown & JSON Spec Export:** ส่งออกข้อกำหนดที่สกัดเสร็จเป็นเอกสารความต้องการเชิงวิศวกรรมซอฟต์แวร์ทันที

## 5. Test-Driven Development (TDD) & UAT Scenarios

### 5.1 Gherkin Test Scenarios

#### Scenario 1: การโหลดและนำเสนอตัวอักษร Kanit ได้สำเร็จ (UAT: Font family loading check)
* **Given** ผู้ใช้เปิดเว็บบราวเซอร์เข้ามายังระบบ IREBA
* **When** ระบบบราวด์เซอร์ทำการเรนเดอร์เอกสารและเปิดสไตล์ชีทหลัก
* **Then** ระบบต้องทำการดาวน์โหลดชุดฟอนต์ `'Kanit'` จากลิงก์ Google Fonts สำเร็จ
* **And** ฟอนต์สำหรับหัวข้อและเนื้อหาทุกตำแหน่งต้องแสดงผลในรูปแบบตัวอักษร Kanit อย่างเป็นระเบียบ

#### Scenario 2: การกดเลขรุ่นแอปพลิเคชันเพื่อขยายแถบรายการฟีเจอร์ (UAT: Click App Version for feature list dropdown)
* **Given** ผู้ใช้งานอยู่ในส่วน Footer ล่างสุดของหน้าจอใดๆ ในระบบ IREBA
* **When** ผู้ใช้งานเคลื่อนเมาส์ไปชี้ที่ข้อความ `"IREBA App Version: v3.0.0"` และคลิกหนึ่งครั้ง
* **Then** หน้าจอต้องเปิดแสดงหน้าต่างขนาดเล็ก (Popover/Modal) โดยไม่มีหน้าจอโหลดค้าง
* **And** ข้อมูลรายการคุณลักษณะ (Feature List) 5 ข้อหลักของรุ่นต้องปรากฏขึ้นอย่างสะอาดตาและอ่านง่าย
* **And** เมื่อผู้ใช้งานคลิกพื้นที่อื่นภายนอกกล่อง หรือกดปุ่มปิด กล่องรายละเอียดต้องปิดตัวลงโดยทันที

## 6. Expected Output Summary (สรุปผลลัพธ์ที่ต้องการ)
1. สไตล์ชีทส่วนกลางที่จัดตั้งตัวแปร CSS Variables ทั้งหมด แยกออกจากตรรกะหน้าจอเพื่อความสะดวกต่อการปรับปรุงเฉดสีและขนาดฟอนต์ภายหลัง
2. ส่วนควบคุม Footer แสดงเลขรุ่นแอปพลิเคชัน IREBA `v3.0.0` พร้อมตรรกะเปิดแสดงกล่องรายการความสามารถแอปพลิเคชัน
