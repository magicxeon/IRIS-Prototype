# Phase 2: ยกระดับหน้า Dashboard และฟังก์ชันกรองข้อมูล (Dashboard Enhancements)

แผนงานขั้นที่สองมุ่งปรับปรุงหน้าแดชบอร์ดโครงการทั้งหมด (Screen 1) ให้มีฟังก์ชันการค้นหา คัดกรองสถานะ และสรุปข้อมูลสถิติจำนวนโครงการสะสมแบบกระชับ โดยย้ายตัวสถิติไปฝังลงในปุ่มตัวเลือกตัวกรองเพื่อสุขลักษณะทางสายตาที่ดีของแถบแสดงข้อมูล (Data Density & Spacing)

---

## 🛠️ รายละเอียดการออกแบบสถาปัตยกรรม (Design Specifications)

### 1. ระบบตัวเลขสะสมในปุ่มตัวกรอง (Inline Pill Metrics)
* ปรับเปลี่ยนรูปแบบสรุปจำนวนโครงการ จากเดิมที่เป็นบอร์ดการ์ดขนาดใหญ่ (ซึ่งเกะกะสายตา) มาเป็น **Badge ขนาดจิ๋วฝังตัวขนานไปกับปุ่มคัดกรองสถานะโครงการ (Pills)**:
  * ปุ่มทั้งหมด: แสดงยอดโครงการรวมสะสมในระบบ
  * ปุ่มรอดำเนินการ: แสดงยอดโครงการที่มีสถานะ `initialized` (รอโหลดเอกสาร) หรือ `documents_uploaded` (รอบังคับวิเคราะห์)
  * ปุ่มวิเคราะห์แล้ว: แสดงยอดโครงการที่มีสถานะ `analyzed` (วิเคราะห์เสร็จสิ้น)
  * ปุ่มอนุมัติแล้ว: แสดงยอดโครงการที่มีสถานะ `approved` (อนุมัติผลลัพธ์แล้ว)
* **กลไกรองรับข้อมูลปริมาณมาก (99+ Limits & Tooltip)**:
  * หากตัวเลขโครงการในสถานะใดก็ตามมีจำนวน **มากกว่า 99 โครงการ** ตัว Badge จะถูกแปลงข้อความเป็น **"99+"** ทันที เพื่อประหยัดพื้นที่ปุ่มและไม่ให้รกสายตา
  * ในขณะเดียวกัน ปุ่มจะถูกฝังด้วย `title` Attribute เพื่อสร้าง Tooltip พื้นบ้านของเบราว์เซอร์ แสดงข้อมูลเต็มเมื่อวางเมาส์ชี้ เช่น `"จำนวนจริง: 105 โครงการ"`

### 2. ระบบค้นหาและการคัดกรองข้อมูล (Client-side Search & Filters)
* ช่องค้นหา: ใช้กล่องค้นหาแบบขอบกลมมน `🔍 ค้นหารหัสหรือชื่อโครงการ...`
* การคัดกรองทำงานแบบ Real-time บนบราวเซอร์ของผู้ใช้ โดยไม่จำเป็นต้องยิง Request ดึงข้อมูลใหม่จากเซิร์ฟเวอร์ทุกครั้งที่พิมพ์ เพื่อความรวดเร็วและตอบสนองได้ทันที (Instant feedback)

### 3. วงกลมอักษรย่อและรหัสสีโครงการ (Project Avatars)
* Prepend วงกลมอวตารข้างหน้าชื่อโครงการในตาราง โดยใช้ตัวย่อพยัญชนะ 2 ตัวแรกที่ดึงจากชื่อโครงการ เช่น โครงการ `"FutureShield Implementation"` -> ได้อักษรย่อ **FS**
* ใช้ฟังก์ชันแฮชรหัสโครงการ (`projectId`) เพื่อสุ่มหาเฉดสีไล่ระดับพรีเมียม (Gradients) ในการกำหนดพื้นหลังวงกลม เพื่อรับประกันว่าโครงการเดิมจะคงแสดงผลสีเดิมเสมอเมื่อหน้าจอมีการรีเฟรช

## 🏗️ แหล่งแก้ไขไฟล์และโค้ดทางปฏิบัติ (Proposed File Changes)

### 1. [index.html](file:///d:/development/IRIS-Training/IRIS-Prototype/public/index.html) (Screen 1)
* ยกเลิกการประกาศแผงการ์ดสถิติขนาดใหญ่ (dashboard-stats-grid)
* อัปเดตโครงสร้างใน `.filter-pills` ให้ปุ่มตัวกรองแต่ละชิ้นบรรจุแท็กสปัน Badge ย่อย:
  * `<span class="pill-count" id="pill-count-all">0</span>`
  * `<span class="pill-count" id="pill-count-pending">0</span>`
  * `<span class="pill-count" id="pill-count-analyzed">0</span>`
  * `<span class="pill-count" id="pill-count-approved">0</span>`

### 2. [style.css](file:///d:/development/IRIS-Training/IRIS-Prototype/public/style.css)
* เอา CSS การ์ดสถิติและกล่องสถิติเดิมออก
* เพิ่มการจัดรูปแบบปุ่มให้แสดงผลแบบ flex และตั้งค่าสไตล์ `.pill-count`:
  * ใส่สีพื้นหลังโปร่งแสงบางเบา (`rgba(15, 23, 42, 0.08)`) และตัวหนังสือสีค่อนข้างเข้มเพื่อให้สะดุดตา
  * เมื่อปุ่มตกอยู่ในสถานะ Active ให้สลับสีของ Badge เป็นสีขาวโปร่งแสงเพื่อให้ตัดกับตัวปุ่มครามอย่างลงตัว

### 3. [app.js](file:///d:/development/IRIS-Training/IRIS-Prototype/public/app.js)
* ดึง DOM Element references ของปุ่ม Badge (`pillCountAll`, `pillCountPending` ฯลฯ)
* สร้างฟังก์ชันควบคุม `formatPillCount(el, count)` เพื่อเช็คขีดจำกัด `99+` และการเขียนค่า Tooltip (`title`)
* อัปเดตฟังก์ชัน `loadProjectsList()` ให้สถิติโครงการที่คำนวณได้ถูกป้อนส่งให้ `formatPillCount` เรนเดอร์บนปุ่มตัวเลือกโดยตรง
* พัฒนาฟังก์ชัน `getProjectInitials(name)` และ `getProjectAvatarColor(projectId)` ช่วยวาดและตกแต่งแถวข้อมูลโครงการ
