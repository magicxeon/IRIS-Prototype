# Phase 2: ยกระดับหน้า Dashboard และฟังก์ชันกรองข้อมูล (Dashboard Enhancements)

แผนงานขั้นที่สองเน้นการเพิ่ม **"สุนทรียภาพแห่งข้อมูล (Data Density & Utility)"** ให้หน้าแดชบอร์ดโครงการทั้งหมดมีความเพียบพร้อม น่าใช้งาน และสะดวกต่อการค้นหาจัดการเฉกเช่นเดียวกับซอฟต์แวร์ระดับ Enterprise

---

## 🛠️ รายละเอียดการเปลี่ยนแปลงของไฟล์ (Proposed Code Changes)

### 1. [index.html](file:///d:/development/IRIS-Training/IRIS-Prototype/public/index.html) (ส่วน Screen 1 - Dashboard View)
* **เพิ่มส่วนประกอบ Statistics Cards**:
  * เพิ่มกลุ่มแท็ก `<div class="dashboard-stats-grid">` ด้านบนสุดของตัวตารางใน `dashboard-view`
  * ประกอบไปด้วยการ์ดสรุปข้อมูลสถิติ 4 ใบ:
    1. **โครงการรวมทั้งหมด (Total Projects)**: แสดงไอคอนแฟ้ม 📂
    2. **รอการวิเคราะห์ (Pending Analysis)**: โครงการที่อยู่ในสถานะ `initialized` หรือ `documents_uploaded` (ไอคอนนาฬิกาทราย ⏳)
    3. **วิเคราะห์แล้ว (Analyzed)**: โครงการที่สกัด AI เสร็จแล้ว (ไอคอนกราฟสเปกตรัม 📊)
    4. **ผ่านการอนุมัติ (Approved)**: โครงการที่มีการรีวิวเสร็จสมบูรณ์ (ไอคอนเครื่องหมายถูกสีเขียว ✅)
* **เพิ่มส่วนควบคุม Search & Filters**:
  * แทรกกล่องปุ่มตัวเลือกและช่องเสิร์ชใต้ส่วน Statistics Cards:
    * ช่องค้นหา: `<input type="text" id="project-search-input" placeholder="🔍 ค้นหารหัสหรือชื่อโครงการ...">`
    * ปุ่มฟิลเตอร์ตัวกรองสถานะ: กลุ่มปุ่มแบบ Pill-toggle (ทั้งหมด | รอดำเนินการ | วิเคราะห์แล้ว | ออนุมัติแล้ว)

### 2. [style.css](file:///d:/development/IRIS-Training/IRIS-Prototype/public/style.css)
* **สไตล์สำหรับการ์ดสถิติ (Stats Grid)**:
  * กำหนด Grid Layout ที่แสดงผลแบบตอบสนอง (Responsive) ลื่นไหล:
    ```css
    .dashboard-stats-grid {
      display: grid;
      grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
      gap: 1.25rem;
      margin-bottom: 2rem;
    }
    ```
  * ออกแบบให้ตัวการ์ดสถิติมีส้นเงาบางเบา (Soft shadow) และมีสีไล่เฉดบางเบาในสัญลักษณ์ไอคอนหลังการ์ด
* **สไตล์ส่วนค้นหาและปุ่มตัวกรอง**:
  * จัดการตำแหน่งให้อยู่ในแถวเดียวกันบน Desktop และตัดแบ่งเมื่อใช้ในจอมือถือ (Responsive wrap)
  * ออกแบบปุ่มตัวกรองสถานะให้ขอบกลมแบบ Pill เปลี่ยนสีสะท้อนเมื่อคลิกเลือกใช้งาน (Active filter)
* **สไตล์ตารางและอวตารวงกลม (Project Initials)**:
  * พัฒนาคลาส `.project-avatar-circle` ขนาด `40px` ที่มีสีพื้นหลังไล่ระดับแบบสุ่ม และแสดงอักษรย่อของโครงการตรงกลาง เช่น **FS** สีขาวเด่น

### 3. [app.js](file:///d:/development/IRIS-Training/IRIS-Prototype/public/app.js)
อัปเกรดโค้ดการโหลดข้อมูลและฟังก์ชันฟิลเตอร์ของแดชบอร์ดโครงการ:
* **ผูกตัวแปรการค้นหาและฟิลเตอร์สถานะ (State Management)**:
  * สร้างสถานะค้นหาภายในหน้า: `let searchTerm = '';` และ `let activeStatusFilter = 'all';`
* **คำนวณจำนวนในการ์ดสถิติแบบ Real-time**:
  * ในฟังก์ชัน `loadProjectsList` ให้อัปเดตตัวเลขแสดงผลบนการ์ด 4 ใบจากอาเรย์ผลลัพธ์ที่ดึงออกมาจาก API โดยตรง
* **สร้างตัวแยกอักษรย่อและแปลงสีสุ่ม (Avatar Generator)**:
  * เพิ่มฟังก์ชันตัวช่วย `getProjectInitials(name)` ในการดึงพยัญชนะ 2 ตัวแรกมาแสดงผล
  * ใช้ฟังก์ชันสีสุ่มคำนวณจากรหัสโครงการ (`projectId`) เพื่อให้โครงการเดิมแสดงผลสีของวงกลมเดิมเสมอเมื่อหน้าจอรีโหลด
* **ปรับปรุงตัวกรองตาราง (Client-side Search & Filter)**:
  * เขียนตรรกะกรองรายการโครงการที่ดึงมาจาก API ด้วยเงื่อนไขค้นหาชื่อ/คำอธิบาย และกรองสถานะที่เลือก ก่อนที่จะรันการเรนเดอร์ลงในตาราง `projectsTableBody`
  * ผูก Event 'input' เข้ากับช่องค้นหา (พร้อมฟังก์ชันหน่วงเวลาเล็กน้อย debouncing เพื่อไม่ให้หน่วงเครื่องขณะพิมพ์เร็ว) และผูก Click event กับปุ่มตัวกรองสถานะ

## 🔍 แผนการตรวจสอบความถูกต้อง (Verification Details)
1. **ตรวจสอบความถูกต้องของระบบนับสถิติ**: ตรวจสอบว่าผลรวมและประเภทโครงการใน Stats card ด้านบนตรงกับแถวข้อมูลโครงการในระบบจริงในไฟล์ `projects.json`
2. **ทดสอบฟังก์ชันการค้นหาและการกรอง**:
   * ทดสอบพิมพ์คำบางส่วนในช่องค้นหา ตารางจะต้องกรองแถวข้อมูลอัปเดตทันที
   * ทดสอบคลิกปุ่มตัวกรองสถานะ เช่น กด "อนุมัติแล้ว" จะต้องมีเฉพาะโครงการที่มีสถานะ `approved` ปรากฏ
3. **ตรวจสอบความคงที่ของสีวงกลมอวตาร**: ตรวจสอบว่าชื่อโครงการเดียวกันคงสีเดิมเสมอ และหน้าตาสวยงามตัดสีข้อความชัดเจน
