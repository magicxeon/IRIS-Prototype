# Phase 5: การปรับปรุงแผงตรวจสอบสเปกด้วยแถบนำทางแนวตั้ง (Vertical Spec Editor Sidebar)

ในสเตปการตรวจสอบสเปกข้อกำหนดประกันภัย (**Screen 4**) เดิมระบบใช้แถบแท็บแนวนอนด้านบนที่เมื่อมีจำนวนหัวข้อมากจะเกิดปัญหาการสไลด์เลื่อน (Scroll) ซึ่งทำให้ภาพลักษณ์ของระบบไม่เป็นมืออาชีพ 

แผงปฏิบัติการขั้นที่ห้านี้เป็นโซลูชันเพื่อยกระดับ UI/UX โดยเปลี่ยนผ่านดีไซน์ไปสู่ **Vertical Sidebar Navigation (แถบแท็บแนวตั้งด้านซ้ายคอลัมน์คู่)** เลียนแบบรูปแบบของโปรแกรมจัดการเอกสารสมัยใหม่ (เช่น Notion, Slack หรือ IDEs) พร้อมฟังก์ชัน **Collapsible Sidebar (ย่นย่อแถบเมนูเพื่อคืนพื้นที่ตาราง)** ซึ่งช่วยให้ผู้ใช้วิเคราะห์ตารางคณิตศาสตร์ประกันภัยที่มีคอลัมน์เยอะๆ ได้อย่างกว้างขวางเต็มจอ

## 🛠️ รายละเอียดการเปลี่ยนแปลงของไฟล์ (Proposed Code Changes)

### 1. [index.html](file:///d:/development/IRIS-Training/IRIS-Prototype/public/index.html)
* **ปรับโครงสร้างตัวครอบและขยายขนาดของการ์ด Screen 4**:
  * ปรับเปลี่ยนสไตล์อินไลน์ `max-width` ของการ์ด `#screen-4` จากเดิม `1100px` ไปเป็น `1200px` เพื่อรองรับหน้าจอที่กว้างขึ้น
  * เพิ่ม Tag ตัวครอบหลัก `<div class="spec-editor-layout">` 
  * ภายใต้แถบแท็บด้านซ้าย ให้เพิ่มปุ่มสลับย่นย่อแถบเมนู (Sidebar Collapse Toggle):
  ```html
  <div class="spec-editor-layout">
    <!-- Left Sidebar for Spec Tabs -->
    <aside class="tabs-bar vertical" id="tabs-bar-s4">
      <!-- ส่วนปุ่มย่อเมนูจะถูกเรนเดอร์โดย JavaScript หรือใส่ไว้ใน HTML ด้านบนสุดของบาร์ -->
      <button class="spec-sidebar-toggle" id="btn-spec-sidebar-toggle" title="ย่อแถบเมนู">◀</button>
      <div class="tab-links-list" id="tab-links-list-s4"></div>
    </aside>

    <!-- Right Content Area for Active Spec Panel -->
    <div class="tab-contents-wrapper" id="tab-contents-container"></div>
  </div>
  ```

### 2. [style.css](file:///d:/development/IRIS-Training/IRIS-Prototype/public/style.css)
* **ปรับปรุงสไตล์โครงผังจัดวาง (CSS Layout)**:
  * **`.spec-editor-layout`**:
    * ใช้การแสดงผลแบบ `display: flex;` กำหนดช่องว่าง `gap: 1.5rem;` และจัดทิศทางแนวระนาบ
  * **`.tabs-bar.vertical`**:
    * เปลี่ยนเป็น `flex-direction: column;` กว้างปกติ `240px` และใช้ `transition: width 0.3s cubic-bezier(0.4, 0, 0.2, 1);` เพื่อการย่อขนาดที่ลื่นไหล
    * เมื่อสวมคลาส `.collapsed` ความกว้างจะลดลงเหลือ **`60px`**
  * **ปุ่มย่อเมนู `.spec-sidebar-toggle`**:
    * ออกแบบปุ่มสไตล์ Minimalist วางชิดขอบหรือขวาบนของบาร์ ขนาด 28px
    * คอยหมุนลูกศร 180 องศาเมื่อถูกพับ: `.collapsed .spec-sidebar-toggle { transform: rotate(180deg); }`
  * **ตัวข้อความในปุ่มแท็บ `.tab-link span`**:
    * ครอบชื่อฟิลด์ภาษาไทยด้วยแท็ก `<span>` เพื่อให้เวลาพับเมนู สามารถซ่อนข้อความได้อย่างสะอาดตา:
      * `.collapsed .tab-link span { display: none; }` หรือซ่อนด้วย `opacity: 0; transition: opacity 0.2s;`
      * ปรับแต่งตัวปุ่มเมื่อย่อเหลือเฉพาะไอคอนตรงกลาง: `.collapsed .tab-link { justify-content: center; padding: 0.85rem; }`
  * **`.tab-contents-wrapper`**:
    * กำหนด `flex: 1;` เพื่อให้หน้าจอเนื้อหาด้านขวาขยายพื้นที่คืนกลับมาโดยอัตโนมัติเมื่อ Sidebar ด้านซ้ายยืดหด
  * **การจัดการแบบ Responsive สำหรับอุปกรณ์หน้าจอเล็ก (< 768px)**:
    * กำหนดซ่อนปุ่มสลับย่นย่อเมนู: `#btn-spec-sidebar-toggle { display: none; }`
    * สลับทิศทางของเลย์เอาต์ `.spec-editor-layout` เป็น `flex-direction: column;` และให้บาร์ `.tabs-bar.vertical` สลับทิศทางเป็นแนวนอนแบบเดิม (`flex-direction: row;` กว้าง 100% พร้อม `overflow-x: auto;`) เพื่อความเข้ากันได้ของการแสดงผลบนหน้าจอโทรศัพท์มือถือ

## 🔍 แผนการตรวจสอบความถูกต้อง (Verification Details)

### Manual Verification
1. สปินอัปเซิร์ฟเวอร์ และเปิดเข้าไปดูสเปกในหน้า Screen 4
2. สังเกตที่หัวมุมบนของแถบแท็บด้านซ้าย จะต้องปรากฏปุ่ม `◀` เล็กๆ
3. เมื่อ **กดปุ่ม `◀`**:
   * แถบ Sidebar จะหดความกว้างลงสู่ 60px อย่างนุ่มนวล
   * ชื่อข้อความภาษาไทยของแท็บจะค่อยๆ จางหายไป เหลือแสดงผลเพียงไอคอนรูปสัญญลักษณ์ (🌐, 📝, 🧮, 🏥, ⚖️) จัดกึ่งกลาง
   * แผงตารางตัวเลขเบี้ยและตารางเวนคืนกรมธรรม์ด้านขวา จะต้องขยายสัดส่วนกว้างขึ้นรับพื้นที่ทั้งหมดในทันที
4. เมื่อ **กดปุ่ม `▶` อีกครั้ง**:
   * แถบ Sidebar จะขยายความกว้างกลับมาเป็น 240px และแสดงป้ายชื่อหัวข้อต่างๆ เช่นเดิมปกติ
