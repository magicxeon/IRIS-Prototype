# Phase 3: เอฟเฟกต์การสลับหน้าและ Visual Refinements (Visual Polish & Transitions)

แผนการดำเนินการในขั้นสุดท้ายเน้นการขัดเกลาและเพิ่ม **"ความหรูหราระดับพรีเมียม (WOW Factor & Refinement)"** ให้กับระบบ ผ่านการทำอนิเมชั่นระดับไมโคร (Micro-interactions) ปรับแต่งหน้าตาตัวเลือก ปุ่ม ช่องแก้ไข และการควบคุมการแสดงผลบนอุปกรณ์เคลื่อนที่

---

## 🛠️ รายละเอียดการเปลี่ยนแปลงของไฟล์ (Proposed Code Changes)

### 1. [style.css](file:///d:/development/IRIS-Training/IRIS-Prototype/public/style.css)
* **เพิ่มการเคลื่อนไหวเมื่อสลับหน้าจอ (Smooth Page Transitions)**:
  * พัฒนาคลาสอนิเมชั่น `.fade-slide-in` เพื่อให้ข้อมูลสลับหน้าไม่เกิดการตัดภาพแบบเฉียบพลัน:
    ```css
    .screen-view {
      animation: screenFadeIn 0.4s cubic-bezier(0.16, 1, 0.3, 1) forwards;
    }
    @keyframes screenFadeIn {
      from {
        opacity: 0;
        transform: translateY(12px);
      }
      to {
        opacity: 1;
        transform: translateY(0);
      }
    }
    ```
* **ปรับปรุงหน้าตาแท็บตรวจสอบผลลัพธ์ (Tabs Design - Screen 4)**:
  * ออกแบบแถบแท็บ `.tabs-bar` ใหม่ให้มีสไตล์ Minimal เรียบหรู:
    * แทนที่ปุ่มธรรมดาด้วยแท็บที่มีเส้นขอบล่างบางๆ (Border-bottom line) และไฮไลท์สีครามเฉพาะแท็บที่เปิดอยู่
    * เพิ่มเส้นแอนิเมชั่นวิ่งตามตำแหน่งแท็บที่คลิก (Active line slider) 
* **ปรับปรุงช่องทางนำเข้าไฟล์ (Dropzones & Success Details - Screen 2)**:
  * เพิ่มความลื่นไหลให้กับคลาส `:hover` ของการ์ดอัปโหลดให้มีการลอยตัวเล็กน้อย (TranslateY)
  * ตอนที่ลากไฟล์มาวางเหนือบ็อกซ์ (Dragover) ให้เปลี่ยนกรอบเส้นประเป็นอนิเมชั่นวิ่งขยับรอบๆ (Animated dashed border)
* **อัปเกรดความพรีเมียมตัว Loading Overlay ตอนส่งวิเคราะห์ AI**:
  * เพิ่มประสิทธิภาพ Glassmorphic backdrop-filter ให้มีความชัดเจน เบลอสวยงามบนบราวเซอร์ทุกค่าย
  * ใช้ CSS Spinner แบบวงกลมทูโทนสว่างวิ่งรอบตัวเอง (Double-ring SVG spinner) แทนการใช้รูปฟันเฟืองอิโมจิ (`⚙️`) เพื่อภาพลักษณ์ที่สะอาดเป็นทางการ

### 2. [app.js](file:///d:/development/IRIS-Training/IRIS-Prototype/public/app.js)
* **การจัดการ Animation Classes ระหว่างสลับหน้า**:
  * ในฟังก์ชัน `showScreen(screenNumber)` ให้ทำหน้าจอที่เรียกแสดงผลรีเซ็ตคลาสแอนิเมชั่น เพื่อให้การเล่นแอนิเมชั่นทางสายตา (CSS animation trigger) เกิดขึ้นทุกครั้งที่มีการเปลี่ยนหน้าจอไม่ใช่แค่ตอนโหลดเว็บบราวเซอร์ครั้งแรก
* **การเพิ่ม Interactive Feedback ในช่องกรอกผลลัพธ์**:
  * ในหน้าจอ Screen 4 ตอนแก้เนื้อหาที่เป็นตารางหรือ Contenteditable ให้ใส่ฟังก์ชันจับเหตุการณ์ (Focus/Blur) เพื่อสลับแสดงความสว่างกรอบสีฟ้า (Focus Outline Ring) ให้ผู้ใช้อ่านรู้ได้ทันทีว่าช่องนี้สามารถพิมพ์แก้ไขได้อยู่ ณ ปัจจุบัน

### 3. [index.html](file:///d:/development/IRIS-Training/IRIS-Prototype/public/index.html) (การตอบสนองบนมือถือ - Responsive Layout)
* **เพิ่มปุ่มแฮมเบอร์เกอร์และแถบสไลด์เปิด/ปิด Sidebar**:
  * เพิ่มบาร์บุ่มเมนูแฮมเบอร์เกอร์ขนาดเล็ก `<button id="sidebar-toggle" class="sidebar-toggle">☰</button>` ทางมุมขวาบน เพื่อใช้ในกรณีหน้าจอมีขนาดความกว้างแคบกว่า `768px` (แท็บเล็ตและมือถือ)
  * เมื่อกดปุ่มดังกล่าว แถบ Sidebar ด้านข้างจะเลื่อนสไลด์ออกมาจากขอบซ้าย (Drawer menu) บังหน้าหลักบางส่วน และสไลด์เก็บเมื่อผู้ใช้เคาะพื้นที่ว่างภายนอก

## 🔍 แผนการตรวจสอบความถูกต้อง (Verification Details)
1. **ทดสอบแอนิเมชั่นเปลี่ยนหน้า**: สังเกตการคลิกเปลี่ยนหน้าจาก Sidebar หรือการกดปุ่ม ย้อนกลับ/ถัดไป หน้าจอจะต้องสไลด์ขึ้นและจางเข้าอย่างนุ่มนวล
2. **ทดสอบการแสดงผลบน Responsive Simulator**:
   * ทดสอบย่อขนาดหน้าจอบราวเซอร์ลงต่ำกว่า `768px` ตรวจสอบว่า Sidebar ยุบเก็บอัตโนมัติ และปุ่ม Hamburger ใช้งานเปิดปิดนำทางได้ถูกต้อง
   * ตารางสูตร Actuarial และ Matrix ในหน้าจอ 4 จะต้องแสดงผลด้วยการมีแถบสกอร์แนวนอนแยกในตัวการ์ด ไม่ล้นดึงให้โครงสร้างบอดี้ฉีกขาดออกจากกัน
3. **ตรวจสอบความละเอียดแสงเงา**: หน้าจอกรอบกล่องข้อความ, ไอคอน, และบาร์ดาวน์โหลดไม่เกิดข้อผิดพลาดในการทับซ้อนกันของ Layer (Z-index Check)
